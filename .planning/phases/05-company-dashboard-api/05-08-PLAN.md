    - Heading: dashboard.buildTest.steps.role

    - When role selected → wizardData.role = role; enable Next button
    - Next button (→ 'modules')

    **Step 2 — Modules (build-test-modules frame):**
    - Step indicator: "Step 2 of 4"
    - Heading: dashboard.buildTest.steps.modules

    - Back button (→ 'role'), Next button (→ 'custom-questions')

    **Step 3 — Custom Questions (build-test-custom-questions frame):**
    - Step indicator: "Step 3 of 4"
    - Heading: dashboard.buildTest.steps.questions
    - List of added custom questions (max 3) — each shows question text + type + delete button
    - "Add Question" button (dashboard.buildTest.actions.addQuestion) — opens add question modal
    - Questions are optional — "Skip" or "Next" both advance to next step
    - Back button (→ 'modules'), Next button (→ 'generate')

    **Add Question Modal (Dialog/Drawer pattern):**
    - useMediaQuery('(max-width: 768px)') → Drawer on mobile, Dialog on desktop
    - Form: question text (Input, required) + question type (Select: Multiple Choice, Open Text, Ranking)
    - Submit: adds to wizardData.customQuestions (if < 3), closes modal
    - If already 3 questions: "Add Question" button is disabled

    **Step 4 — Generate (build-test-generate frame):**
    - Step indicator: "Step 4 of 4"
    - Heading: dashboard.buildTest.steps.generate
    - Summary card: role, selected modules (as badges), custom question count
    - "Generate Test" button (dashboard.buildTest.actions.generate):
      - Sets generating=true, shows spinner/loading state on button
      - POST /api/tests with body: { role: wizardData.role, name: `${wizardData.role} Assessment`, modules: wizardData.modules, customQuestions: wizardData.customQuestions }
      - On success: setNewTestId(response.id), setStep('preview'), setGenerating(false)
      - On error: toast.error(dashboard.errors.loadFailed), setGenerating(false)
    - Back button (→ 'custom-questions')

    **Preview step (build-test-preview frame):**
    - Read-only summary of the test with fake questions list (e.g. "12 questions across 3 modules")
    - "Looks Good → Continue" button (→ 'success')
    - Back button (→ 'generate')

    **Success modal (build-test-success frame):**
    - When step === 'success': renders as Dialog (cannot be dismissed by ESC — use open={true} without onOpenChange)
    - Title: dashboard.buildTest.successTitle (h2 + checkmark icon)
    - Body: dashboard.buildTest.successBody
    - "Copy Test Link" button:
      - Constructs URL: `${window.location.origin}/test/${newTestId}/intro`
      - navigator.clipboard.writeText(url) + toast.success(dashboard.buildTest.copyLinkSuccess)
    - "Send to Candidates" button:
      - Opens mailto: `mailto:?subject=Test+Invitation&body=...`
    - "View Test" button: router.push('/dashboard/tests/' + newTestId)
    - The wizard resets: setStep('role'), setWizardData defaults
