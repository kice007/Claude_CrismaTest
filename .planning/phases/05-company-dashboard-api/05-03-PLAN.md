  <action>
    Create src/app/api/tests/route.ts with two exported handlers:

    **GET /api/tests:**
    - Supabase query: `supabase.from('test_templates').select('id, role, slug, name, modules, duration, status, created_at, response_count').order('created_at', { ascending: false })`
    - Support optional query params: `?status=` (filter by status), `?search=` (ilike on name or role)
    - Returns array of test template summaries
    - Error: `{ error: 'Internal server error' }` with 500

    **POST /api/tests:**
    - Accepts JSON body: `{ role: string, name: string, modules: string[], customQuestions?: { text: string, type: string }[] }`
    - Validate with zod: role and name required (non-empty strings), modules array min 1, customQuestions optional array max 3 items
    - Generate shareable_link: `${process.env.NEXT_PUBLIC_APP_URL}/test/${crypto.randomUUID()}/intro` (or a slug-based URL)
    - Insert into test_templates: `{ role, name, modules, custom_questions: customQuestions ?? [], status: 'active', shareable_link, response_count: 0 }`
    - Return inserted row with 201 status
    - Validation error: `{ error: 'Invalid fields' }` with 400
    - Server error: `{ error: 'Internal server error' }` with 500

    Import pattern: `import { NextRequest, NextResponse } from 'next/server'` + `import { createServerSupabaseClient } from '@/lib/supabase'` + `import { z } from 'zod'`

    Use `export async function GET(req: NextRequest)` and `export async function POST(req: NextRequest)` — both exported from the same file.

    Create src/app/api/tests/[id]/route.ts with three exported handlers:

    **GET /api/tests/[id]:**
    - Supabase query: fetch from test_templates by id, then join questions:
      ```typescript
      const { data: test } = await supabase
        .from('test_templates')
        .select('*, questions(*)')
        .eq('id', id)
        .single()
      ```
    - If not found: `{ error: 'Not found' }` with 404
    - Returns full test object with nested questions array and shareable_link

    **PUT /api/tests/[id]:**
    - Accepts JSON body: `{ name?: string, modules?: string[], customQuestions?: { text: string, type: string }[] }`
    - Validate with zod: all fields optional but at least one must be present (use .refine())
    - Supabase update: `supabase.from('test_templates').update({ name, modules, custom_questions }).eq('id', id).select().single()`
    - If not found: `{ error: 'Not found' }` with 404
    - Returns updated test object with 200

    **DELETE /api/tests/[id]:**
    - Soft delete: update status to 'archived' (do NOT hard-delete rows)
    - Supabase: `supabase.from('test_templates').update({ status: 'archived' }).eq('id', id)`
    - Verify row exists first — if not found: `{ error: 'Not found' }` with 404
    - Return 204 No Content on success (empty body)

    Route params pattern for Next.js App Router:
    ```typescript
    export async function GET(
      req: NextRequest,
      { params }: { params: Promise<{ id: string }> }
    ) {
      const { id } = await params
      // ...
    }
    ```
    Apply the same `{ params }` destructure pattern to PUT and DELETE handlers.

    All handlers: try/catch wrapping, console.error('[tests/[id]]', error.message), minimal error responses.
