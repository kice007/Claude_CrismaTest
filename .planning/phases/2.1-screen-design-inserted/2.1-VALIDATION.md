# Phase 2.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Visual verification via mcp__pencil__get_screenshot + structural via mcp__pencil__batch_get |
| **Config file** | None — Pencil MCP provides all verification tooling |
| **Quick run command** | `mcp__pencil__get_screenshot` on current frame after creation |
| **Full suite command** | `mcp__pencil__batch_get` on each .pen file to confirm all expected frame IDs exist |
| **Estimated runtime** | ~5–15 seconds per frame screenshot; ~30s full file verification |

---

## Sampling Rate

- **After every task commit:** Run `mcp__pencil__get_screenshot` on the last frame created
- **After every plan wave:** Run `mcp__pencil__batch_get` on all completed .pen files to confirm frame IDs
- **Before `/gsd:verify-work`:** All 6 .pen files present in `/design/`, all frame counts verified
- **Max feedback latency:** ~15 seconds per frame
