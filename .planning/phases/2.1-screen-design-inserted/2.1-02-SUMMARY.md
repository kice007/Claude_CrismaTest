## Files Created/Modified

## Decisions Made

- **Atomic commit for both tasks:** The .pen JSON format requires a complete valid document — same pattern established in Plan 01. Tasks 1 and 2 written atomically.
- **NavShell not rebuilt:** Auth pages use full-screen centered card layout (no nav) — the imports array documents the cross-file reference for completeness, but auth frames themselves include a note annotation confirming NavShell is intentionally omitted.
- **forgot-password is post-submit confirmation only:** AUTH-03 specifies a static confirmation screen. No email input field in this frame — Phase 4 implementers handle the email request form separately.
- **Error state frames are full copies:** sign-up--error and login--error duplicate the base frame structure with error styling applied. This makes the error states immediately legible without needing overlays or annotations to explain what changed.
- **Google OAuth 'Coming soon' as note:** Note annotation on OAuth button in sign-up and login frames captures the AUTH-01 requirement without building a separate toast frame. Phase 4 implementers will wire the toast in code.
