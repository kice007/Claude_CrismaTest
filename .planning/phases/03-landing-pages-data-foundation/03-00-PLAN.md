  <how-to-verify>
    **Step 1 — Create Supabase project (if not already done):**
    1. Go to https://supabase.com and sign in (or create free account)
    2. Click "New Project"
    3. Name it: `crismatest` (or similar)
    4. Choose a region close to you
    5. Set a database password (save it somewhere safe)
    6. Wait for project to provision (~1 minute)
    7. Go to Project Settings → API
    8. Copy: **Project URL**, **anon/public key**, **service_role key**

    **Step 2 — Install npm packages:**
    ```bash
    npm install @supabase/ssr resend zod react-hook-form
    ```
    Verify: `npm list @supabase/ssr resend zod react-hook-form` shows all four packages.

    **Step 3 — Create .env file (NOT .env.local, NOT committed):**
    Create `.env` in the project root with these 6 variables:
    ```
    NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...your-anon-key...
    SUPABASE_SERVICE_ROLE_KEY=eyJ...your-service-role-key...
    RESEND_API_KEY=re_...your-resend-key...
    NEXT_PUBLIC_APP_URL=http://localhost:3000
    ADMIN_EMAIL=your@email.com
    ```

    For RESEND_API_KEY: go to https://resend.com, create account (free tier), get API key from Dashboard → API Keys.

    **Step 4 — Verify .env is not tracked by git:**
    ```bash
    git status
    ```
    `.env` must NOT appear in the output. If it does, run: `git rm --cached .env`
