<objective>
Build the two primary auth entry points: the multi-step company sign-up flow (/sign-up) and the login page (/login). Sign-up uses AnimatePresence step transitions through 5 states (form → otp → onboarding-1 → onboarding-2 → complete). Login is a single-step form. Both use react-hook-form + standardSchemaResolver + zod. Both call setAuthSession() on success and redirect to /dashboard.

Purpose: AUTH-01 and AUTH-02 — the two main auth entry points users encounter from the home page CTA buttons.
Output: Two page.tsx files inside the (auth) route group.
</objective>

**Visual design:** Full-screen centered card layout. White card (max-w-md) on a brand-light (#EEF2FF) background. Logo top of card. No NavShell (handled by route group layout).

**Step machine:**
```typescript
type SignUpStep = 'form' | 'otp' | 'onboarding-1' | 'onboarding-2' | 'complete'
const [step, setStep] = useState<SignUpStep>('form')
const [email, setEmail] = useState('')  // passed to OTP subtitle interpolation
```

**AnimatePresence transitions:** Wrap each step in `<AnimatePresence mode="wait">` with a keyed `motion.div` using `fadeIn` variant (initial="hidden" animate="visible" exit="hidden") — quick 200ms transition between steps. Import `{ AnimatePresence, motion }` from `motion/react`.

**Step 1 — 'form' (sign-up form):**
Fields (react-hook-form + standardSchemaResolver + zod):
- companyName: z.string().min(1)
- firstName: z.string().min(1)
- lastName: z.string().min(1)
- email: z.string().email()
- password: z.string().min(8)
- companySize: z.string().min(1) — `<select>` with 4 options from i18n keys
- country: z.string().min(1) — text input (not a full country select in v1; placeholder "e.g. France")

Show/hide password toggle: eye icon button (use `Eye` / `EyeOff` from lucide-react), toggles `<input type="password" | "text">`.

Google OAuth button (top of card, above form): `onClick={() => toast.info(t('auth_google_coming_soon'))}` — does NOT navigate.

Submit handler: `setEmail(data.email); setStep('otp')` — no real network call.

Use `t('auth_signup_*')` keys for all labels, placeholders, CTAs. Error messages via `t('auth_field_required')`, `t('auth_invalid_email')`, `t('auth_password_min')`.

"Already have an account? Log in" link → `/login` via Next.js `<Link>`.

**Step 2 — 'otp' (modal overlay):**
Show OTP verification UI on top of the blurred form (or replace the card content — simpler for v1).
Title: `t('auth_otp_title')`. Subtitle: `t('auth_otp_subtitle', { email })`.
Single 6-digit input (type="text" maxLength={6}). Any 6-digit code advances (no real validation).
CTA: `onClick={() => setStep('onboarding-1')}`.
Resend link (static, shows `toast.success('Code resent!')` — uses `t('auth_otp_resend')`).

**Step 3 — 'onboarding-1':**
Left context text (or top of card): `t('auth_onboarding_step', { current: 1, total: 2 })` badge.
Title: `t('auth_onboarding_1_title')`. Subtitle: `t('auth_onboarding_1_subtitle')`.
Two fields:
- industry: `<select>` with options (Technology, Finance, Healthcare, Retail, Other)
- hiringVolume: `<select>` with options (1–5, 6–20, 21–50, 51+)
CTA `t('auth_onboarding_1_cta')` → `setStep('onboarding-2')`.

**Step 4 — 'onboarding-2':**
Step badge: `t('auth_onboarding_step', { current: 2, total: 2 })`.
Title: `t('auth_onboarding_2_title')`. Subtitle: `t('auth_onboarding_2_subtitle')`.
Role chip multi-select (static array: Customer Support, Sales, Marketing, Engineering, Finance, HR, Operations, Other). Chips are toggleable with brand-primary selected state. At least 1 required.
CTA `t('auth_onboarding_2_cta')` → `setStep('complete')`.

**Step 5 — 'complete' (all-set modal content):**
Centered card. Checkmark circle icon (brand-primary, 64px). Title: `t('auth_allset_title')`. Subtitle: `t('auth_allset_subtitle')`.
CTA button: `t('auth_allset_cta')` → calls `setAuthSession()` then `router.push('/dashboard')`.
Import `setAuthSession` from `@/lib/auth`. Import `useRouter` from `next/navigation`.

**Input styling:** Consistent across all steps. `border border-slate-200 rounded-lg px-3 py-2 w-full focus:outline-none focus:ring-2 focus:ring-brand-primary`. Error state: `border-red-500`. Error message: `<p className="text-sm text-red-500 mt-1">{errors.field?.message}</p>`.
