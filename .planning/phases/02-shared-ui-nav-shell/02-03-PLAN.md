    Update src/app/layout.tsx to add the Toaster component. The Toaster must be placed as a SIBLING of `<I18nProvider>{children}</I18nProvider>`, NOT nested inside it. It goes AFTER the I18nProvider closing tag, still inside MotionConfig:

    ```tsx
    import type { Metadata } from "next";
    import { Inter, JetBrains_Mono } from "next/font/google";
    import { MotionConfig } from "motion/react";
    import { I18nProvider } from "@/components/I18nProvider";
    import { Toaster } from "@/components/ui/sonner";
    import "./globals.css";

    // ... font declarations unchanged ...

    export default function RootLayout({
      children,
    }: Readonly<{
      children: React.ReactNode;
    }>) {
      return (
        <html lang="en" className={`${inter.variable} ${jetbrainsMono.variable}`}>
          <body className="antialiased">
            <MotionConfig reducedMotion="user">
              <I18nProvider>{children}</I18nProvider>
              <Toaster
                position="bottom-right"
                visibleToasts={3}
                richColors
                closeButton
                duration={4000}
              />
            </MotionConfig>
          </body>
        </html>
      );
    }
    ```

    Critical placement rules:
    - Toaster is a SIBLING of I18nProvider, not a child — toasts render in their own portal and must not be inside the i18n context wrapper
    - Toaster stays INSIDE MotionConfig so it inherits the reducedMotion="user" setting
    - Do NOT nest Toaster inside I18nProvider — this causes focus trap conflicts when a Dialog is open (see RESEARCH.md Pitfall 4)
    - Preserve all existing metadata, font declarations, and html/body attributes unchanged

    Props explanation:
    - position="bottom-right" — satisfies CONTEXT decision "bottom-right is typical for this pattern"
    - visibleToasts={3} — max 3 stacked simultaneously; 4th triggers oldest auto-dismiss
    - richColors — applies semantic colors (green=success, amber=warning, red=error)
    - closeButton — adds × dismiss button on each toast (per CONTEXT decision)
    - duration={4000} — 4 second auto-dismiss (per UI-03 spec)
