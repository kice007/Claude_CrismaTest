How deep should the redesign go?

Three approaches for applying the Bold/Data-heavy style to the homepage. My recommendation is highlighted.

A

Surgical Updates (2–3 sections)

Change only the highest-impact surface areas: white hero with left-aligned headline + badge, section labels with decorative flankers, dark navy CTA block redesign. The middle content (steps, features, test library) stays as-is.

✓ Fastest ✓ Lower risk ✗ Inconsistent feel mid-page
B

Full Redesign ⭐ Recommended (all sections)

Rebuild every section with Datashake patterns: white hero with left text + right product UI, decorated section labels throughout, problem/solution framing, 5-column feature stats row, large quote testimonial block, dark navy CTA with data grid texture. Cohesive end-to-end.

Sections to redesign:
🔄 Hero → white, left-aligned, badge 🔄 Trust bar → cleaner logo row ➕ "The Problem" section 🔄 Steps → labeled feature sections 🔄 Features → icon stat grid 🔄 Testimonials → large quote block 🔄 Final CTA → dark navy + grid bg ✓ Footer → keep as-is
✓ Cohesive, polished result ✓ True Datashake feel ✗ More implementation work
C

Hero + CTA Only (first + last impression)

Update only the hero (white bg, badge, left-aligned) and the final CTA block (dark navy + grid texture). Everything in between stays exactly as designed. Best if you want to test the feel quickly before committing.

✓ Quickest preview ✗ Most inconsistent result
My recommendation: Option B — Full Redesign. You've picked a strong design direction; applying it only to part of the page will feel incomplete. The homepage is a single scroll, so visual consistency across all sections matters. We can work section by section, so the effort is manageable.
